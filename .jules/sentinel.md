## 2026-07-10 - Downstream HTTP Parameter/Path Injection in Public API Proxies
**Vulnerability:** The repository parser `parseGithubRepoUrl` allowed lazy wildcard matching `[^/\s]+?` for owner/repo fields, which could match query parameters, hash fragments, and authority delimiters (like `@` or `?` or `#`). This allowed callers of the public `/api/github-meta` proxy to manipulate the upstream fetch URL requested by the server.
**Learning:** Naive regular expressions used for URL parsing on the backend can inadvertently capture URL control components (like query parameters or hash segments) into path variables. This can lead to downstream HTTP request injection or Server-Side Request Forgery (SSRF) risks when those components are concatenated into subsequent `fetch` calls.
**Prevention:** Always sanitize input URLs by stripping query strings and hash fragments *before* extracting path variables. Additionally, validate the extracted variables against strict whitelisted character sets matching the upstream provider's naming specifications (e.g., standard GitHub alphanumeric, hyphen, underscore, and period characters).
